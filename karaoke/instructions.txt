Deploy without a VM
cd c:\pers\krishnaposa-mlo-site\azure\functions\karaoke-func\infra
.\deploy.ps1 -ResourceGroupName "rg-karaoke-prod" -Location "canadacentral"
No -SshPublicKey needed. After deploy, get the storage connection string (portal or az storage account show-connection-string) and point local_worker.py at it (same names as today: karaoke-input / karaoke-output / karaoke-status / karaoke-jobs).

local.settings.json now includes WORKER_VM_ENABLED": "false" so local Functions behave the same way.

If you ever want the VM path
.\deploy.ps1 -ResourceGroupName "rg-karaoke-prod" -DeployWorkerVm `
  -SshPublicKey (Get-Content "$env:USERPROFILE\.ssh\id_ed25519.pub" -Raw)
That sets WORKER_VM_ENABLED=true on the Function App and creates the VM, network, and role assignment.

Code touchpoints
infra/main.bicep: param deployWorkerVm bool = false; VM/network/role only when true; app setting WORKER_VM_ENABLED.
shared.py: worker_vm_enabled(); ensure_vm_running / deallocate_vm no-op when disabled.
vmreaper/init.py: returns immediately when the worker VM path is disabled.
infra/deploy.ps1: -DeployWorkerVm optional; validates SSH key only when that switch is used.
