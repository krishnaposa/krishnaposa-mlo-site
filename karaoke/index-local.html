Karaoke Split — local folder queue

Same flow as the main page: upload one mixed audio file (vocals + band together). This page talks to a small Python server in this repo—not Azure Functions on your machine (func start is not used). Your file is saved into an input folder as the queue; a background thread runs Spleeter/Demucs and writes stems under an output folder; the player loads those files over HTTP.

Run: python karaoke/local_folder_queue.py (defaults: KARAOKE_LOCAL_ROOT = ~/.karaoke-local, port 8787, SEPARATOR=spleeter). Override base URL: ?api=http://127.0.0.1:8787 or set window.KARAOKE_API_BASE before scripts load. For the hosted cloud pipeline, use Karaoke Split (cloud).

HTTPS blocks local submit. This page is loaded over https://, so the browser will not call http://127.0.0.1:8787 (mixed content)—Submit will show “Failed to fetch”. Open this file over http:// instead, for example: from the repo root run python -m http.server 8765, then go to http://localhost:8765/karaoke/index-local.html (adjust path if needed). Or open index-local.html via file:// from disk.
